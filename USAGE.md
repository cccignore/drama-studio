# Drama Studio · 使用指南

> 这份文档是给**创作者**看的，不是给程序员看的。
> 只要你能打开一个网页、会复制粘贴，就能用它写出一部 5 集短剧。

在线地址：<http://43.160.246.165/>

---

## 一分钟速通

1. 打开网址
2. 首页点 **「进入工作台」**
3. 点 **「新建项目」**，随便起个名字
4. 在第一步 `start` 里，点 **「一键填入 5 集试玩案例」**
5. 接下来 **一路点「下一步」** 就行，每一步都会自动生成内容
6. 写完任意一集，就可以点 **复盘** 给这集打分
7. 最后点 **导出**，拿到 Word 文档

整个流程大概需要 5–10 分钟，取决于模型快慢。

---

## 界面长什么样

工作台左边是一条 7 步进度条：

```
立项 → 节奏规划 → 角色 → 分集目录 → 分集剧本 → 复盘 → 导出
start   plan      characters  outline  episode     review  export
```

你的当前位置会高亮。可以随时**点左边的步骤名字跳回去**改之前的内容，不会丢数据。

右上角有个 **「项目增强」** 按钮，里面是进阶玩法（出海模式、多模型切换、多角色协同），普通使用**可以不管**。

## 每一步在做什么

### 1. 立项（start）

告诉 AI 你想写什么：题材、主角人设、卖点。

- **不确定怎么写？** 点「一键填入 5 集试玩案例」，会自动填入一个"霸总重生"样例。
- **想用语音？** 输入框右边有麦克风，口述就行（需要浏览器允许麦克风权限）。
- **想改？** 直接改输入框里的文字，再点「生成」。

### 2. 节奏规划（plan）

AI 会给你一份"前 30 集节奏曲线"——什么时候出钩子、什么时候卡付费点。

这一步**不用你写任何东西**，看一眼确认节奏合理，点「下一步」即可。

### 3. 角色 & 关系图（characters）

AI 生成主要角色设定，并画一张关系图。

- 可以拖动、缩放关系图。
- 如果觉得某个人设不对，可以回到立项页补充描述，再重跑这一步。

### 4. 分集目录（outline）

AI 会把整部剧拆成若干集，每一集有标题、主线、钩子、爽点。

- 🔥 = 爽点（观众看到想拍桌子）
- 💰 = 付费卡点（这里断更/卡付费最有效）

### 5. 分集剧本（episode）

**这是你最常用的一步。**

- 左边选集数（第 1 集、第 2 集……）
- 右边点「生成这一集」
- AI 会吐出带场次、台词、镜头提示、音乐提示的完整剧本

剧本格式：

```
## 场 1 · 咖啡馆（外滩 / 日）

△ （特写）林夏推门而入
♪ 轻柔钢琴曲
**林夏**（惊讶）：你怎么会在这里？
**陆辰**：我等了你三年。
```

- `△` = 镜头/动作
- `♪` = 音乐
- `**名字**（情绪）：台词` = 对白

### 6. 复盘（review）

每写完一集，都可以让 AI 给这一集打 5 维分：

- **节奏** / **爽感** / **台词** / **格式** / **连贯**

还会告诉你这一集有哪些**具体问题 + 修改建议**（比如"第 3 场对话太平，加一个冲突"）。

右边有个**雷达图**，一眼看出这集哪方面弱。

### 7. 导出

三种格式，按需要下载：

| 格式 | 什么时候用 |
| --- | --- |
| **Markdown (.md)** | 想继续用 VS Code / Obsidian 编辑 |
| **Word (.docx)** | 要交给制片、导演、平台 |
| **完整工程 (.zip)** | 想留档或者发给同事完整打包 |

## 进阶玩法（项目增强面板）

右上角「项目增强」按钮，里面分四块：

### 🌏 出海模式

- 一键切换到**中文场记 + 英文对白**的双语剧本格式
- 适合给海外平台或者海外制作方
- 切回国内模式也是一个按钮的事

### ✅ 合规检查

- 对已写剧本跑一遍风险扫描
- 输出**红/黄/绿三色面板**：红=可能过不了审、黄=有风险、绿=没问题
- 每条都会告诉你该怎么改

### 🎚 多模型切换（MoE）

- 默认推荐**质量优先 MoE**：GPT-5.4 做结构，DeepSeek 写长剧本，Grok 做审校
- 这里可以**给不同步骤绑不同模型**，也可以在模型设置页修改全局角色槽位
- 换掉 Secondary 槽位后，所有项目的剧本写作模型都会跟着变

### 🧠 多角色协同

- 打开后，AI 会分裂成三个角色共同创作：
  - **Planner**（策划）先搭骨架
  - **Critic**（编审）挑毛病
  - **Writer**（编剧）根据编审意见改写
- 产出质量更稳，但速度慢 2–3 倍、也更费 token
- 可以只在**节奏规划**或**分集剧本**上打开

## 常见问题

### Q1. 生成卡住不动？

- 右上角控制台会显示"正在生成…"。模型响应时间在 10s–3min 之间，耐心等。
- 如果超过 5 分钟还没动静，刷新页面重试，数据不会丢。

### Q2. 生成的内容不满意怎么办？

- **每一步**右上角都有「重新生成」按钮，点一下会再跑一次。
- 每个已生成产物旁边还有「编辑」按钮，可以让 AI 按你的指令局部改写，也可以手动改 Markdown 源码，并随时从历史版本回滚。
- 或者回到上一步补充描述（例如在立项里写明"主角要有海外留学背景"），再重跑下游。

### Q3. 我在第 3 集了，能不能回去改第 1 集？

- 可以。点左边进度条的 `episode` → 选第 1 集 → 重新生成。
- **但注意**：重跑第 1 集后，第 2、3 集不会自动更新，需要你手动重跑下游。

### Q4. 项目列表里可以开几个项目？

- 不限。项目之间**完全隔离**，互不影响数据。

### Q5. ⚠️ 模型 API key 是我一个人用，还是所有访客共享？

**共享。**当前部署没有账号系统。任何打开这个网址的人，都能看到你配置的模型列表、并用同一个 key 发起生成请求，**会消耗你的 API 额度**。

- key 本身在数据库里是**加密存储**的，UI 里只显示 `sk-***abcd` 这种掩码，别人**看不到完整 key**。
- 但他们可以**使用**这个 key（通过正常的生成流程）。
- 如果想保护额度，有两个办法：
  1. 只把网址分享给信任的人
  2. 在服务器 nginx 上加一道 Basic Auth

如果部署者使用云雾网关，可以在服务器 `.env` 填 `YUNWU_API_KEY`，执行 `npm run init-db` 后会自动预置 GPT-5.4 / DeepSeek-V3.2 / Grok-4.2 三条模型，并启用质量优先 MoE 角色槽位。

### Q6. 写的项目会不会被别人看到？

会。同样因为没有账号系统，**所有访客共享一个项目列表**。如果只给你自己用，别把网址发出去。

### Q7. 浏览器语音输入没反应？

- Chrome / Edge 支持最好，Safari 部分型号有兼容问题。
- 第一次用要允许"使用麦克风"权限。
- 如果还是不行，直接打字就行。

### Q8. 页面刷新会丢数据吗？

不会。所有生成结果都存在服务器的数据库里，刷新、关掉浏览器、甚至换设备打开（只要记得项目 ID）都能继续。

## 一条完整的创作路径示例

> 假设你要写一部 5 集"重生复仇"短剧：

1. 打开 <http://43.160.246.165/>
2. 新建项目「重生之三年后」
3. `start`：填入"女主意外穿越回三年前，带着未来记忆避开前夫背叛"。
4. `plan`：看 AI 给的节奏曲线，确认付费卡点放在第 3 集。
5. `characters`：检查女主、前夫、白月光三个角色设定。关系图点开看一眼。
6. `outline`：AI 拆出 5 集目录，每集都有 🔥 和 💰 标记。
7. `episode`：**第 1 集**生成 → **复盘 & 打分** → 发现"台词"只有 6 分，回去重跑一次。
8. 重复第 2、3、4、5 集。
9. `export`：下载 Word，发给制片。

整个过程对创作者基本是**"点点按钮 + 看看结果"**，真正的写作交给 AI。
